## 2. Vuex状态管理

### 模块结构

```
store/modules/
├── app.js        // 应用设置 (侧边栏、主题、多页签)
├── user.js       // 用户状态 (token、信息、权限列表)
├── permission.js // 路由权限 (动态路由生成)
└── online.js     // 在线用户状态
```

### user.js State

```javascript
{
  token: '',           // 登录token
  username: '',        // 用户名
  realname: '',        // 真实姓名
  permissionList: [],  // 权限列表
  info: {}             // 用户信息
}
```

### app.js State

```javascript
{
  sidebar: { opened: true },  // 侧边栏状态
  device: 'desktop',         // 设备类型
  theme: 'dark',             // 主题
  layout: 'side',            // 布局类型
  multipage: true            // 多页签模式
}
```

## 3. API交互模式

### 请求封装 (src/utils/request.js)

```javascript
// 基于axios封装，配置了请求拦截器和响应拦截器
// BaseURL: /ctrl (开发环境代理到 http://xxx.xxx.xxx.xxx:20000)

// GET请求
getAction(url, params)

// POST请求
postAction(url, data)

// PUT请求
putAction(url, data)

// DELETE请求
deleteAction(url, params)
```

### API模块划分 (src/api/)

```
api/
├── system/
│   ├── user.js      // 用户管理API
│   ├── role.js      // 角色管理API
│   ├── permission.js // 权限API
│   ├── department.js // 部门API
│   ├── dict.js      // 字典API
│   └── log.js       // 日志API
├── apply/
│   ├── application.js // 应用API
│   ├── service.js    // 服务API
│   └── process.js    // 流程API
├── document/
│   └── document.js   // 文档API
└── gateway/
    └── gateway.js    // 网关API
```

### 统一响应格式

```javascript
  "result": { /* 数据 */ },
  "code": 200
}
```

## 4. 路由配置

### 路由加载流程

```
src/router/index.js
    ↓
src/store/modules/permission.js
    ↓ (动态加载路由)
getRouters() -> 过滤权限 -> 生成菜单
```

### 路由守卫

```javascript
// 前置守卫
router.beforeEach((to, from, next) => {
  // 验证登录状态
  // 检查权限
  // 更新标签页
})
```

### 路由结构

```javascript
{
  path: '/system',
  component: Layout,
  redirect: '/system/users',
  children: [
    { path: 'users', component: UserList },
    { path: 'roles', component: RoleList },
    // ...
  ]
}
```

## 5. 核心页面操作流程

### 用户管理操作流程

```
列表页 → [查询] → [添加] → 弹出表单 → 填写信息 → [保存]
                 ↓
          [编辑] → 弹出表单 → 修改信息 → [保存]
                 ↓
          [删除] → 确认弹窗 → [确认删除]
                 ↓
          [冻结] → API调用 → 状态更新
```

### 流程编排器操作流程

```
拖拽节点到画布 → 配置节点属性 → 连接节点 → 保存流程
                                    ↓
                              [发布] → 流程生效
```
