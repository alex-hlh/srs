## 二、日常运维

### 2.1 健康检查

```bash
#!/bin/bash
# health_check.sh - 系统健康检查脚本

echo "======================================"
echo "星环空间 - 健康检查"
echo "时间: $(date '+%Y-%m-%d %H:%M:%S')"
echo "======================================"

# 1. 检查后端服务
echo -e "\n[1] 检查后端服务..."
BACKEND_SERVICES=("srs-center:8081" "srs-ctrl:8082" "srs-gateway:8080")

for service in "${BACKEND_SERVICES[@]}"; do
    IFS=':' read -r name port <<< "$service"
    if curl -s -o /dev/null -w "%{http_code}" http://localhost:$port/actuator/health | grep -q "200"; then
        echo "  ✓ $name [OK]"
    else
        echo "  ✗ $name [FAIL]"
    fi
done

# 2. 检查中间件
echo -e "\n[2] 检查中间件..."

# MySQL
if mysqladmin ping -h localhost -u root -p${MYSQL_ROOT_PASSWORD} --silent 2>/dev/null; then
    echo "  ✓ MySQL [OK]"
else
    echo "  ✗ MySQL [FAIL]"
fi

# Redis
if redis-cli -h localhost -p 6379 ping | grep -q PONG; then
    echo "  ✓ Redis [OK]"
else
    echo "  ✗ Redis [FAIL]"
fi

# 3. 检查端口
echo -e "\n[3] 检查端口监听..."
PORTS=(3306 6379 9876 9200 9000 8080 8081 8082)
for port in "${PORTS[@]}"; do
    if lsof -i:$port > /dev/null 2>&1; then
        echo "  ✓ 端口 $port [监听中]"
    else
        echo "  ✗ 端口 $port [未监听]"
    fi
done

# 4. 磁盘检查
echo -e "\n[4] 检查磁盘空间..."
df -h | awk '$5 > 80 {print "  ⚠ " $0}'

# 5. 内存检查
echo -e "\n[5] 检查内存使用..."
free -h | awk '/^Mem:/ {printf "  总量: %s 已用: %s 使用率: %.0f%%\n", $2, $3, ($3/$2)*100}'

echo -e "\n======================================"
echo "检查完成"
echo "======================================"
```

### 2.2 日志管理

**日志目录结构:**

```
/var/log/ft/
├── srs-center/                 # 认证中心日志
│   ├── application.log        # 应用日志
│   ├── error.log             # 错误日志
│   └── access.log            # 访问日志
├── srs-ctrl/                   # 管控平台日志
│   ├── application.log
│   ├── error.log
│   └── sql.log               # SQL 日志
├── srs-gateway/                # 网关日志
│   ├── application.log
│   └── route.log             # 路由日志
├── nginx/                     # Nginx 日志
│   ├── access.log
│   └── error.log
└── elasticsearch/             # ES 日志
    └── gc.log
```

**日志查看命令:**

```bash
# 查看实时日志
tail -f /var/log/ft/srs-ctrl/application.log

# 查看错误日志
tail -100 /var/log/ft/srs-ctrl/error.log | grep ERROR

# 搜索关键字
grep -r "Exception" /var/log/ft/srs-ctrl/error.log

# 按时间范围查询
sed -n '/2024-01-15 10:00:00/,/2024-01-15 11:00:00/p' application.log

# 日志压缩归档
find /var/log/ft -name "*.log" -mtime +7 -exec gzip {} \;

# 清理 30 天前的日志
find /var/log/ft -name "*.gz" -mtime +30 -delete
```

### 2.3 备份策略

| 数据类型 | 备份频率 | 保留时间 | 存储位置 |
|----------|----------|----------|----------|
| 数据库 | 每日全量 | 30天 | 本地 + OSS |
| Redis RDB | 每小时 | 7天 | 本地 |
| 文件存储 | 每日增量 | 30天 | 本地 + OSS |
| 日志 | 每日 | 90天 | 日志服务器 |

**备份脚本:**

```bash
#!/bin/bash
# backup.sh - 数据备份脚本

set -e

BACKUP_DIR=/data/backup
DATE=$(date +%Y%m%d)
MYSQL_ROOT_PASSWORD=your_password

# 创建备份目录
mkdir -p ${BACKUP_DIR}/${DATE}

# 1. 备份 MySQL
echo "备份 MySQL 数据库..."
docker exec ft-mysql mysqldump \
    -uroot -p${MYSQL_ROOT_PASSWORD} \
    --single-transaction \
    --routines \
    --triggers \
    srs_db | gzip > ${BACKUP_DIR}/${DATE}/mysql_${DATE}.sql.gz

# 2. 备份 Redis
echo "备份 Redis 数据..."
docker exec ft-redis redis-cli SAVE
docker cp ft-redis:/data/dump.rdb ${BACKUP_DIR}/${DATE}/redis_${DATE}.rdb

# 3. 备份 MinIO 数据
echo "备份 MinIO 数据..."
mc mirror minio/xx-ft ${BACKUP_DIR}/${DATE}/minio/

# 4. 备份 Nginx 配置
echo "备份 Nginx 配置..."
cp -r /etc/nginx ${BACKUP_DIR}/${DATE}/nginx_config

# 5. 清理旧备份 (保留 30 天)
find ${BACKUP_DIR} -type d -mtime +30 -exec rm -rf {} \;

# 6. 上传到 OSS (可选)
# ossutil cp -r ${BACKUP_DIR}/${DATE} oss://your-bucket/ft-backup/

echo "备份完成: ${DATE}"
```

## 四、故障处理

### 4.1 服务无法启动

```bash
# 1. 检查端口占用
lsof -i:8082

# 2. 检查日志
tail -100 /var/log/ft/srs-ctrl/error.log

# 3. 检查内存
free -h

# 4. 常见问题
# - 端口被占用: kill -9 <PID> 或修改端口
# - 内存不足: 调整 JVM -Xmx 参数
# - 数据库连接失败: 检查 MySQL 服务和防火墙
```

### 4.2 数据库连接池耗尽

```bash
# 1. 查看当前连接数
mysql -u root -p -e "SHOW PROCESSLIST;"

# 2. 查看连接池状态
curl http://localhost:8082/actuator/druid

# 3. 解决方式
# - 重启应用释放连接
systemctl restart srs-ctrl
# - 杀掉长期占用的连接
mysql -u root -p -e "KILL <PROCESS_ID>;"
```

### 4.3 Redis 连接失败

```bash
# 1. 检查 Redis 服务
systemctl status redis
redis-cli ping

# 2. 检查内存
redis-cli info memory | grep used_memory_human

# 3. Redis 满了处理
# - 调整 maxmemory
redis-cli config set maxmemory 2gb
# - 清理过期 key
redis-cli CLUSTER FSLUSHALL
# - 重启 Redis
systemctl restart redis
```

### 4.4 服务雪崩应对

```
                    正常请求              故障时
用户请求 ──────────▶ 应用 ──────────▶ 数据库
                              │
                              ▼
                         [超时/异常]

处理策略:
1. 限流: nginx limit_req
2. 熔断: Sentinel / Resilience4j
3. 降级: 返回缓存数据或默认值
4. 扩容: 增加服务实例
```

**Hystrix 降级配置:**

```java
@HystrixCommand(
    fallbackMethod = "getUserFallback",
    commandProperties = {
        @HystrixProperty(name = "execution.isolation.thread.timeoutInMilliseconds", value = "3000"),
        @HystrixProperty(name = "circuitBreaker.requestVolumeThreshold", value = "20"),
        @HystrixProperty(name = "circuitBreaker.sleepWindowInMilliseconds", value = "5000")
    }
)
public User getUser(String id) {
    return userService.getById(id);
}

public User getUserFallback(String id) {
    // 返回缓存数据或默认值
    return cacheService.getUser(id).orElse(new User());
}
```

## 五、性能优化

### 5.1 JVM 调优

```bash
# 推荐 JVM 参数
JAVA_OPTS="
  -Xms2g                      # 初始堆大小
  -Xmx2g                      # 最大堆大小
  -XX:+UseG1GC               # 使用 G1 垃圾收集器
  -XX:MaxGCPauseMillis=200   # 最大 GC 停顿时间
  -XX:+HeapDumpOnOutOfMemoryError  # OOM 时生成堆转储
  -XX:HeapDumpPath=/var/log/ft/heapdump.hprof
  -XX:+PrintGCDetails
  -Xloggc:/var/log/ft/gc.log
  -Dspring.profiles.active=prod
"
```

### 5.2 数据库优化

```sql
-- 查看慢查询
SHOW VARIABLES LIKE 'slow_query_log%';
SHOW VARIABLES LIKE 'long_query_time';

-- 优化表
OPTIMIZE TABLE sys_user;

-- 查看索引使用
EXPLAIN SELECT * FROM sys_user WHERE username = 'admin';

-- 添加索引
CREATE INDEX idx_username ON sys_user(username);
CREATE INDEX idx_create_time ON sys_user(create_time);
```

### 5.3 Nginx 优化

```nginx
# worker 进程数
worker_processes auto;

# 单 worker 进程最大连接数
worker_connections 10240;

# 开启 gzip
gzip on;
gzip_types text/plain text/css application/json application/javascript;

# 连接超时
keepalive_timeout 65;

# 缓存静态资源
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=ft_cache:10m max_size=1g;
```

## 六、应急响应

### 6.1 服务故障应急流程

```
1. 发现故障 (监控告警/用户反馈)
       │
       ▼
2. 初步评估
   - 影响范围
   - 故障等级 (P0/P1/P2)
   - 预计恢复时间
       │
       ▼
3. 紧急处理
   - P0: 立即回滚 / 切换服务
   - P1: 30分钟内处理
   - P2: 2小时内处理
       │
       ▼
4. 问题定位
   - 查看日志
   - 检查监控
   - 复现问题
       │
       ▼
5. 修复验证
   - 本地测试
   - 预发布验证
       │
       ▼
6. 正式发布
   - 灰度发布
   - 监控指标
       │
       ▼
7. 故障复盘
   - 编写故障报告
   - 制定改进措施
```

### 6.2 回滚操作

```bash
# 1. 查看历史版本
docker images | grep srs-ctrl

# 2. 回滚到上一版本
docker stop srs-ctrl
docker rm srs-ctrl
docker run -d --name srs-ctrl \
    -p 8082:8082 \
    srs-ctrl:previous_version

# 3. 回滚数据库 (慎用)
mysql -u root -p srs_db < /data/backup/YYYYMMDD/mysql_backup.sql
```
