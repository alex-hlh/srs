## 1. JDK 升级方案 (8 -> 21)

### 1.1 兼容性分析

| 特性 | JDK 8 | JDK 17 | JDK 21 | 备注 |
|------|-------|--------|--------|------|
| 语言特性 | 完整 | sealed class, pattern matching | virtual threads, pattern matching | JDK 21 推荐 |
| Spring Boot 3.x 支持 | 不支持 | 3.0-3.2 | 3.2+ | 必须 JDK 17+ |
| 性能 | - | ZGC, G1 | ZGC, Virtual Threads | JDK 21 更优 |
| 长期支持 | 2030+ | 2026+ | 2031+ | JDK 21 更长 |

**推荐**: JDK 21 (LTS, Virtual Threads 生产可用)

### 1.2 pom.xml 修改

```xml
<!-- 当前 -->
<maven.compiler.source>8</maven.compiler.source>
<maven.compiler.target>8</maven.compiler.target>
<maven.compiler.version>1.8</maven.compiler.version>

<!-- 修改为 -->
<maven.compiler.source>21</maven.compiler.source>
<maven.compiler.target>21</maven.compiler.target>
<maven.compiler.version>21</maven.compiler.version>
```

**涉及文件**:
- srs-platform/pom.xml
- srs-center/pom.xml
- srs-ctrl/pom.xml
- srs-gateway/pom.xml
- srs-doc-center/pom.xml
- srs-meeting/pom.xml
- common/base/pom.xml
- common/component/pom.xml
- common/utils/pom.xml

### 1.3 Lombok 版本升级

```xml
<lombok.version>1.18.30</lombok.version>
```

### 1.4 建议 JVM 参数

```bash
--add-opens java.base/java.lang=ALL-UNNAMED
--add-opens java.base/java.util=ALL-UNNAMED
--add-opens java.base/java.lang.reflect=ALL-UNNAMED
```

## 2. Spring Boot 升级方案 (2.5.4 -> 3.2.x)

### 2.1 Jakarta EE 9+ 迁移 (javax.* -> jakarta.*)

**影响范围**: 81 个 Java 文件

**自动化迁移方案**:

```bash
# 添加 rewrite-spring 插件
<plugin>
    <groupId>org.openrewrite</groupId>
    <artifactId>rewrite-spring</artifactId>
    <version>5.0.0</version>
</plugin>

mvn rewrite:spring-boot-3
```

**手动迁移点**:

```java
// javax.servlet -> jakarta.servlet
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;

// javax.validation -> jakarta.validation
import jakarta.validation.constraints.NotNull;
import jakarta.validation.constraints.Size;
```

### 2.2 配置变更点

```yaml
# Spring Boot 3.x 配置变更
spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
  jackson:
    serialization:
      write-dates-as-timestamps: false
```

## 3. 关键依赖升级路径

### 3.1 版本兼容矩阵

| 依赖 | 当前版本 | 推荐目标 | 风险 |
|------|---------|---------|------|
| MyBatis-Plus | 3.5.2 | 3.5.5 | 低 |
| Druid | 1.2.5 | 1.2.20 | 中 |
| Redisson | 3.13.6 | 3.25.2 | 中 |
| RocketMQ | 4.9.1 | 5.1.4 | 高 |
| Elasticsearch | 7.17.6 | 7.17.6 | 中 |
| dynamic-datasource | 3.2.0 | 4.1.0 | 中 |

### 3.2 RocketMQ 升级注意

RocketMQ 5.x 有重大 API 变更:

```java
// 4.x
org.apache.rocketmq.client.producer.DefaultMQProducer

// 5.x 替代
org.apache.rocketmq.client.MQProducer
```

**建议**: 保持 4.9.1 或使用 spring-boot-starter:

```xml
<dependency>
    <groupId>org.apache.rocketmq</groupId>
    <artifactId>rocketmq-spring-boot-starter</artifactId>
    <version>2.2.3</version>
</dependency>
```

### 3.3 Elasticsearch 升级注意

`RestHighLevelClient` 在 7.17+ 已标记 `@Deprecated`，8.x 移除。

**建议**: 保持 7.17.6 或迁移到新客户端:

```java
// 新客户端 (ES 8.x)
import co.elastic.clients.elasticsearch.ElasticsearchClient;
```
